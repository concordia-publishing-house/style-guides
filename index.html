Commits

Release Notes are generated automatically from our commit messages. For this reason our messages start with tags and read like sentences.

Tags

Commits always begin with a tag wrapped in square braces:

git commit -m "[feature] ..."
Tag Meaning Shown to Customers
[feature] An entirely new feature has been added to the project
[improvement] An existing feature has been improved or enhanced
[fix] A flaw with an existing feature has been fixed
[refactor] The structure of the source code has been changed, but its behavior has not
[testfix]
[cifix]		 Don't mention this commit in a release
[skip] Don't mention this commit in a release

Messages

Commit messages:

  • Read like a sentence that assumes the author of the commit as its subject
  • Begin with a past-tense verb
  • Do not end with a period

git commit -m "[fix] Prohibited unauthorized users from reading the launch codes"

Tickets

Commit messages that resolve a ticket include the ticket number wrapped in square braces:

git commit -m "[improvement] Added a progress indicator for long-running tasks [#63]"

Time

Commit messages end with a record of how much time was spent on the commit:

git commit -m "[feature] Added the ability to determine if any program halts (13h)"
git commit -m "[skip] Deleted a few lines of unused code (2m)"

Author

When pair-programming, commits should be credited to both pairs:

ep pair page lewis
git commit -m "[feature] Gave users the option to have tacos delivered directly to them (4h)"

Source Code

General conventions for source code

White Space

Every file ends with a trailing line break:

class Post
  # ...
end

Source code is indented with soft tabs (2 spaces).

Naming

Temporary Variables

When using a function like inject or reduce, name the temporary variable for its significance (order_total, query) rather than its role in the calculation (memo, result) or its data type (n, hash):

# good
line_items.inject(0) { |balance, line_item| balance += line_item.amount }

# bad
line_items.inject(0) { |memo, line_item| memo += line_item.amount }

Ruby

Casing

Local variables, instance variables, and methods are snake cased:

@instance_variable = :good

def method_name
end

Classes and modules are title cased:

module ActiveRecord
  class Base
  end
end

Constants are screaming snake cased:

ST_DEFAULT_MAX_DENSITY = 5

Nothing is camel cased.

Indentation

Keywords that control method visibility (private, protected, public) are outdented:

class SomeClass

  def public_method
    # ...
  end

private
  
  def private_method
    # ...
  end
end

CoffeeScript

Casing

Local variables, instance variables, and function names are camel cased:

localVariable = 'good'

functionName = ->
  console.log 'hi'

Classes and modules are title cased:

@App ?= {}
@App.Views ?= {}
class @App.Views.ClassName
  # ...

Constants are screaming snake cased:

ST_DEFAULT_MAX_DENSITY = 5

Nothing is snake cased.
